SplitHomogeneousDirectProduct(dimtot, dim1, S)
==============================================

:viewcode:`gdirprod.spad line 130 <gdirprod.spad#L130>` :editcode:`[edit on github] <gdirprod.spad#L130>`

* dimtot: \ :ref:`NonNegativeInteger <l4e6f6e4e65676174697665496e7465676572>`\ 

* dim1: \ :ref:`NonNegativeInteger <l4e6f6e4e65676174697665496e7465676572>`\ 

* S: \ :ref:`OrderedAbelianMonoid <l4f7264657265644162656c69616e4d6f6e6f6964>`\ 


This type represents the finite direct or cartesian product of an underlying ordered component type. The vectors are ordered as if they were split into two blocks. The \ ``dim1``\  parameter specifies the length of the first block. The ordering is lexicographic between the blocks but acts like \ :ref:`HomogeneousDirectProduct <l486f6d6f67656e656f757344697265637450726f64756374>`\  within each block. This type is a suitable third argument for \ :ref:`GeneralDistributedMultivariatePolynomial <l47656e6572616c44697374726962757465644d756c746976617269617465506f6c796e6f6d69616c>`\ .
