NumericalOrdinaryDifferentialEquations
======================================

:viewcode:`numode.spad line 1 <numode.spad#L1>` :editcode:`[edit on github] <numode.spad#L1>`


This package is a suite of functions for the numerical integration of an ordinary differential equation of \ :code:`n`\  variables:   \centerline{dy/dx = \ :code:`f`\ (\ :code:`y`\ , \ :code:`x`\ ) \ :code:`y`\  is an \ :code:`n`\ -vector}   \par All the routines are based on a 4-th order Runge-Kutta kernel. These routines generally have as arguments: \ :code:`n`\ , the number of dependent variables; \ :code:`x1`\ , the initial point; \ :code:`h`\ , the step size; \ :code:`y`\ , a vector of initial conditions of length \ :code:`n`\  which upon exit contains the solution at \ :code:`x1 + h`\ ; \ :code:`derivs`\ , a function which computes the right hand side of the ordinary differential equation: \ :code:`derivs(dydx, y, x)`\  computes \ :code:`dydx`\ , a vector which contains the derivative information.   \par In order of increasing complexity: \begin{items}   \item \ :code:`rk4(y, n, x1, h, derivs)`\  advances the solution vector to \ :code:`x1 + h`\  and return the values in \ :code:`y`\ .   \item \ :code:`rk4(y, n, x1, h, derivs, t1, t2, t3, t4)`\  is the same as \ :code:`rk4(y, n, x1, h, derivs)`\  except that you must provide 4 scratch arrays \ :code:`t1`\ -\ :code:`t4`\  of size \ :code:`n`\ .   \item Starting with \ :code:`y`\  at \ :code:`x1`\ , \ :code:`rk4f(y, n, x1, x2, ns, derivs)`\  uses \ :code:`ns`\  fixed steps of a 4-th order Runge-Kutta integrator to advance the solution vector to \ :code:`x2`\  and return the values in \ :code:`y`\ . Argument \ :code:`x2`\ , is the final point, and \ :code:`ns`\ , the number of steps to take.   \item \ :code:`rk4qc(y, n, x1, step, eps, yscal, derivs)`\  takes a 5-th order Runge-Kutta step with monitoring of local truncation to ensure accuracy and adjust stepsize. The function takes two half steps and one full step and scales the difference in solutions at the final point. If the error is within \ :code:`eps`\ , the step is taken and the result is returned. If the error is not within \ :code:`eps`\ , the stepsize is decreased and the procedure is tried again until the desired accuracy is reached. Upon input, a trial step size must be given and upon return, an estimate of the next step size to use is returned as well as the step size which produced the desired accuracy. The scaled error is computed as \centerline{\ :code:`error = MAX(ABS((y2steps(i) - y1step(i))/yscal(i)))`\ } and this is compared against \ :code:`eps`\ . If this is greater than \ :code:`eps`\ , the step size is reduced accordingly to \centerline{\ :code:`hnew = 0.9 * hdid * (error/eps)^(-1/4)`\ } If the error criterion is satisfied, then we check if the step size was too fine and return a more efficient one. If \ :code:`error`\  > \ :code:`eps`\  * (6.0E-04) then the next step size should be \centerline{\ :code:`hnext`\  = 0.9 * hdid * (\ :code:`error`\ /\ :code:`eps`\ )^(\ :code:`-1/5`\ )} Otherwise \ :code:`hnext = 4.0 * hdid`\  is returned. A more detailed discussion of this and related topics can be found in the book "Numerical Recipes" by \ :code:`W`\ .Press, \ :code:`B`\ .\ :code:`P`\ . Flannery, \ :code:`S`\ .A. Teukolsky, \ :code:`W`\ .\ :code:`T`\ . Vetterling published by Cambridge University Press. Argument \ :code:`step`\  is a record of 3 floating point numbers \ :code:`(to_try , did , next)`\ , \ :code:`eps`\  is the required accuracy, \ :code:`yscal`\  is the scaling vector for the difference in solutions. On input, \ :code:`step.to_try`\  should be the guess at a step size to achieve the accuracy. On output, \ :code:`step.did`\  contains the step size which achieved the accuracy and \ :code:`step.next`\  is the next step size to use.   \item \ :code:`rk4qc(y, n, x1, step, eps, yscal, derivs, t1, t2, t3, t4, t5, t6, t7)`\  is the same as \ :code:`rk4qc(y, n, x1, step, eps, yscal, derivs)`\  except that the user must provide the 7 scratch arrays \ :code:`t1-t7`\  of size \ :code:`n`\ .   \item \ :code:`rk4a(y, n, x1, x2, eps, h, ns, derivs)`\  is a driver program which uses \ :code:`rk4qc`\  to integrate \ :code:`n`\  ordinary differential equations starting at \ :code:`x1`\  to \ :code:`x2`\ , keeping the local truncation error to within \ :code:`eps`\  by changing the local step size. The scaling vector is defined as \centerline{\ :code:`yscal(i) = abs(y(i)) + abs(h*dydx(i)) + tiny`\ } where \ :code:`y(i)`\  is the solution at location \ :code:`x`\ , \ :code:`dydx`\  is the ordinary differential equation\ :code:`'s`\  right hand side, \ :code:`h`\  is the current step size and \ :code:`tiny`\  is 10 times the smallest positive number representable. The user must supply an estimate for a trial step size and the maximum number of calls to \ :code:`rk4qc`\  to use. Argument \ :code:`x2`\  is the final point, \ :code:`eps`\  is local truncation, \ :code:`ns`\  is the maximum number of call to \ :code:`rk4qc`\  to use. \end{items}
