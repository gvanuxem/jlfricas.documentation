ZeroDimensionalSolvePackage(R, ls, ls2)
=======================================

:viewcode:`zerodim.spad line 642 <zerodim.spad#L642>` :editcode:`[edit on github] <zerodim.spad#L642>`

* R: Join(\ :ref:`OrderedRing <l4f72646572656452696e67>`\ , \ :ref:`PolynomialFactorizationExplicit <l506f6c796e6f6d69616c466163746f72697a6174696f6e4578706c69636974>`\ , \ :ref:`RealConstant <l5265616c436f6e7374616e74>`\ )

* ls: \ :ref:`List <l4c697374>`\  \ :ref:`Symbol <l53796d626f6c>`\ 

* ls2: \ :ref:`List <l4c697374>`\  \ :ref:`Symbol <l53796d626f6c>`\ 


A package for computing symbolically the complex and real roots of zero-dimensional algebraic systems over the integer or rational numbers. Complex roots are given by means of univariate representations of irreducible regular chains. Real roots are given by means of tuples of coordinates lying in the \ :ref:`RealClosure <l5265616c436c6f73757265>`\  of the coefficient ring. This constructor takes three arguments. The first one \ :code:`R`\  is the coefficient ring. The second one \ :code:`ls`\  is the list of variables involved in the systems to solve. The third one must be \ :code:`concat(ls, s)`\  where \ :code:`s`\  is an additional symbol used for the univariate representations. WARNING: The third argument is not checked. All operations are based on triangular decompositions. The default is to compute these decompositions directly from the input system by using the \ :ref:`RegularChain <l526567756c6172436861696e>`\  domain constructor. The lexTriangular algorithm can also be used for computing these decompositions (see the \ :ref:`LexTriangularPackage <l4c6578547269616e67756c61725061636b616765>`\  package constructor). For that purpose, the operations \ :ref:`univariateSolve <l5a65726f44696d656e73696f6e616c536f6c76655061636b616765-756e6976617269617465536f6c7665>`\ , \ :ref:`realSolve <l5a65726f44696d656e73696f6e616c536f6c76655061636b616765-7265616c536f6c7665>`\  and \ :ref:`positiveSolve <l5a65726f44696d656e73696f6e616c536f6c76655061636b616765-706f736974697665536f6c7665>`\  admit an optional argument.   Author: Marc Moreno Maza.

  \ :code:`triangSolve(lp, info?, lextri?)`\  decomposes the variety associated with \ :code:`lp`\  into regular chains. Thus a point belongs to this variety iff it is a regular zero of a regular set in in the output. Note that \ :code:`lp`\  needs to generate a zero-dimensional ideal. If \ :code:`lp`\  is not zero-dimensional then the result is only a decomposition of its zero-set in the sense of the closure (with respect to Zariski topology). Moreover, if \ :code:`info?`\  is \ :code:`true`\  then some information is displayed during the computations. See \ :ref:`zeroSetSplit <l526567756c6172547269616e67756c617253657443617465676f7279-7a65726f53657453706c6974>`\ (\ :code:`lp`\ , \ :code:`true`\ , \ :code:`info?`\ ). If \ :code:`lextri?`\  is \ :code:`true`\  then the lexTriangular algorithm is called from the \ :ref:`LexTriangularPackage <l4c6578547269616e67756c61725061636b616765>`\  constructor (see \ :ref:`zeroSetSplit <l4c6578547269616e67756c61725061636b616765-7a65726f53657453706c6974>`\ (\ :code:`lp`\ , \ :code:`false`\ )). Otherwise, the triangular decomposition is computed directly from the input system by using the \ :ref:`zeroSetSplit <l526567756c6172436861696e-7a65726f53657453706c6974>`\  from \ :ref:`RegularChain <l526567756c6172436861696e>`\ .
