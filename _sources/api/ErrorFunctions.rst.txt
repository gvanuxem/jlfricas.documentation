ErrorFunctions
==============

:viewcode:`error.spad line 1 <error.spad#L1>` :editcode:`[edit on github] <error.spad#L1>`


ErrorFunctions implements error functions callable from the system interpreter. Typically, these functions would be called in user functions. The simple forms of the functions take one argument which is either a string (an error message) or a list of strings which all together make up a message. The list can contain formatting codes (see below). The more sophisticated version takes two arguments where the first argument is the name of the function from which the error was invoked and the second argument is either a string or a list of strings, as above. When you use the one argument version in an interpreter function, the system will automatically insert the name of the function as the new first argument. Thus in the user interpreter function \ :code:`f x == if x < 0 then error "negative argument" else x`\  the call to error will actually be of the form \ :code:`error("f","negative argument")`\  because the interpreter will have created a new first argument.   Formatting codes: error messages may contain the following formatting codes (they should either start or end a string or else have blanks around them): \ :code:`\%l`\  start a new line \ :code:`\%b`\  start printing in a bold font (where available) \ :code:`\%d`\  stop printing in a bold font (where available) \ :code:` \%ceon`\  start centering message lines \ :code:`\%ceoff`\  stop centering message lines \ :code:`\%rjon`\  start displaying lines "ragged left" \ :code:`\%rjoff`\  stop displaying lines "ragged left" \ :code:`\%i`\  indent following lines 3 additional spaces \ :code:`\%u`\  unindent following lines 3 additional spaces \ :code:`\%xN`\  insert \ :code:`N`\  blanks (eg, \ :code:`\%x10`\  inserts 10 blanks)
