JLObjPy¶

jobject.spad line 1050 [edit on github]

Convenience domain for Julia Python objects that are used within Julia, and not directly by the underlying FriCAS Common LISP. The PythonCall package has to be installed in Julia. Note that the PythonCall Julia package uses a similar scheme for Julia Python objects to the one of jlFriCAS for JLObjectType references.

jlPyApply: (String, String, %) -> %

jlPyApply(mod, func, pyobj) applies the Python function func from the module mod to the Python object pyobj. For example: Numpy needs to be installed, see the Julia package CondaPkg. example{jlPyApply(“numpy”, “array”, jpyobj(“[1,2,3]”))} creates a numpy array from the Python list [1,2,3]. jlPyApply(“math”, “sqrt”, 2) computes the square root of 2.
